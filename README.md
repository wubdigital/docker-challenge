# 🐳 אתגר הדוקר

## ברוכים הבאים

לפניכם פרויקט נוד שעובד מצוין. הקוד עצמו תקין, אל תיגעו בו.
אבל החבאתי בתוכו כמה בעיות, והן מסתתרות דווקא בהגדרות של הדוקר.

המשימה שלכם, למצוא את הבעיות, להבין מה גורם להן, ולתקן.
עד שהפרויקט נבנה, רץ, ועולה לאוויר.

---

## למה זה חשוב

זה בדיוק מה שמחכה לכם בעולם האמיתי.
בראיונות עבודה נותנים לכם הרבה פעמים בדיוק את זה, מערכת שבורה, ומבקשים שתתקנו.
לא בודקים אם שיננתם פקודות בעל פה. בודקים איך אתם חושבים כשמשהו לא עובד.

אז אל תפחדו משגיאות.
שגיאה אדומה היא לא כישלון, היא רמז. היא אומרת לכם בדיוק מה לא בסדר ואיפה לחפש.
כל באג שתפתרו היום הופך אתכם למתכנתים טובים ובטוחים יותר.

קחו את הזמן. תקנו באג אחד בכל פעם. ותרגישו את ההבדל כשהפרויקט קם לחיים בזכותכם.
אתם יכולים את זה. 🚀

---

## מה צריך להשיג

ראשון, להצליח לבנות אימג' מה-Dockerfile.
שני, להריץ קונטיינר ולראות את האפליקציה עונה בדפדפן.
שלישי, לוודא שקובץ הסודות לא נכנס לאימג'.
רביעי, להעלות את האימג' לדוקר האב.

בונוס, להרים גם מסד נתונים יחד עם האפליקציה, בעזרת קומפוז.

טיפ אחד, תמיד תקראו את הודעת השגיאה לפני שאתם מנחשים.

---

## פקודות שימושיות

לבנות אימג':

```bash
docker build -t docker-challenge .
```

להריץ קונטיינר:

```bash
docker run -p 8080:3000 docker-challenge
```

לראות מה רץ, ולקרוא לוגים:

```bash
docker ps
docker logs <שם הקונטיינר>
```

לנקות קונטיינרים תקועים:

```bash
docker rm -f $(docker ps -aq)
```

להעלות לדוקר האב:

```bash
docker login
docker tag docker-challenge <שם-המשתמש>/docker-challenge
docker push <שם-המשתמש>/docker-challenge
```

להרים הכל יחד עם קומפוז:

```bash
docker compose up --build
docker compose down
```
